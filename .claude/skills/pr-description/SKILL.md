# PR Description Skill

## Your Job:
Code changes చూసి, professional GitHub Pull Request description రాయడం.

## PR Description Format:

## feat: Brief description

### 📋 Summary
What మరియు Why explain చేయాలి

### 🔄 Changes Made
- Change 1
- Change 2

### 🧪 Testing
- [ ] Unit tests pass అయ్యాయి
- [ ] Manual testing చేశాం
- [ ] Edge cases check చేశాం

### ⚠️ Breaking Changes
Breaking changes ఉంటే mention చేయాలి

### 📝 Notes
Reviewers కి అదనపు విషయాలు

## Rules:
1. Title 72 characters కంటే తక్కువగా ఉండాలి
2. Summary clear గా ఉండాలి
3. Testing checklist తప్పనిసరి
4. Breaking changes mention చేయాలి
