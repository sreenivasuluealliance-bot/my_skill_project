# PR Description Skill

## Your Job:
Code changes చూసి, professional GitHub Pull Request description రాయడం.

## PR Description Format:

### Title:
[Type]: Brief description of changes

### Body:
## 📋 Summary
What మరియు Why — ఈ PR లో ఏమి చేశామో, ఎందుకు చేశామో

## 🔄 Changes Made
- Change 1
- Change 2
- Change 3

## 🧪 Testing
- [ ] Unit tests pass అయ్యాయి
- [ ] Manual testing చేశాం
- [ ] Edge cases check చేశాం

## 📸 Screenshots (if applicable)
UI changes ఉంటే screenshots add చేయండి

## ⚠️ Breaking Changes
ఏమైనా breaking changes ఉంటే ఇక్కడ mention చేయండి

## 📝 Notes
Reviewers కి తెలియాల్సిన అదనపు విషయాలు

## Rules:
1. Title 72 characters కంటే తక్కువగా ఉండాలి
2. Summary clear మరియు concise గా ఉండాలి
3. అన్ని changes list చేయాలి
4. Testing checklist తప్పనిసరి
5. Breaking changes తప్పనిసరిగా mention చేయాలి

## Types:
- feat     --> కొత్త feature
- fix      --> bug fix
- docs     --> documentation
- refactor --> code restructure
- test     --> tests add చేసినప్పుడు
